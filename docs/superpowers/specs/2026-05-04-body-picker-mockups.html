Option A — Coarse regions
  • Anatomical line-art front + back, 87 individual muscle paths grouped into 22 region classes
  • One tap on any muscle group selects its parent region (one selection per body area)
  • Maps 1:1 to current BodyLocation enum — zero data-model change
  • Hover surfaces region name; selected regions glow teal across all member muscles
  • Three fallback chips: Spread · Whole body · Not sure

Option C — Two-layer
  • Same anatomical line-art. Tap a region → drawer with muscle subgroups for that area
  • User can tap any number of muscles, or close drawer (✕) to keep just the region tag
  • Adds a BodyMuscle enum + parent-region link to the data model
  • Educational: even glancing at the muscle list teaches anatomy in user's own language

How they differ at a glance

Both replace the chip grid with anatomical line-art bodies — that's the shared visual win. The fork is what happens after the first tap: A treats the region as the answer; C treats it as a doorway into muscle-level detail.
