# M3.4.1 — mediCalm Front Home Page UX/UI Implementation Contract
**Target Environment:** Claude Code Sonnet 4.6  
**Artifact Type:** Markdown build contract for implementation  
**Status:** Active implementation brief  
**Authority Level:** High-priority design and experience contract for the mediCalm front home page

---

## 1. Mission

M3.4.1 exists to transform mediCalm’s front home page from a functional intake screen into a **high-trust, modern, medically grounded, psychologically calming entry experience**.

The page must not feel like:
- a blank medical terminal
- a generic wellness or meditation app
- a basic symptom tracker
- a static intake form

The page must feel like:
- an **intelligent physiological interface**
- a **precision-calming system**
- a **professional medical-adjacent product**
- a **real-time guided relief environment**
- a **modern interface with subtle artistic depth and subconscious trust cues**

This page is not merely a landing page.  
It is the **first nervous-system contact layer** of mediCalm.

Its job is to move the user through:

**Recognition -> Relief -> Action**

before cognitive overload, doubt, or disengagement can occur.

---

## 2. Core Product Truth

The mediCalm homepage must reflect the real mission:

> The user arrives in active bodily discomfort, protective tension, uncertainty, or pain, and needs immediate confidence that this system understands what is happening and can guide them into relief.

The front page must therefore communicate, both consciously and subconsciously:

1. **You are understood**
2. **This system is structured and trustworthy**
3. **This is medically realistic and grounded**
4. **Relief can begin immediately**
5. **You do not need to figure this out alone**
6. **Nature, openness, and decompression are part of healing**
7. **This system is more advanced than a generic breathing app**

---

## 3. Non-Negotiable Identity Pillars

The design must preserve all four simultaneously.

### 3.1 Modern / Ahead of Its Time
The interface should feel current or slightly ahead of current mainstream health-tech UI.  
It should feel intentional, premium, and deeply considered.

### 3.2 Professional / Medically Grounded
The interface must feel credible and realistic in expectations.  
No exaggerated healing claims.  
No childish softness.  
No vague spiritual ambiguity.  
No “spa app” energy.

### 3.3 Calming
The UI must reduce threat and bracing.  
Everything should soften the user’s internal state:
- typography
- spacing
- motion
- color
- interaction tempo
- background depth
- wording

### 3.4 Guided and Personalized
The user must feel that the system is calibrating to their current state, not pushing a one-size-fits-all breathing routine.

---

## 4. Experience Objective

The homepage must accomplish the following within seconds:

### 4.1 Recognition
The user must feel:
> “This understands what I’m dealing with.”

### 4.2 Relief
The interface must already begin downshifting the user’s state before they press the primary button.

### 4.3 Action
The CTA must feel like entering relief, not entering work.

---

## 5. Emotional / Psychological Design Standard

This page must be built on a **subconscious UX standard**, not just visual attractiveness.

The page should quietly signal:
- safety
- precision
- structure
- legitimacy
- spaciousness
- decompression
- flow
- environmental healing
- official value
- modern system intelligence

The user should not consciously narrate these qualities.

## 6. Signature Visual Philosophy

mediCalm should develop a recognizable visual stamp similar in principle to the subtle authority cues found in official design systems such as currency engraving, document micro-patterning, certificates, or secure printed materials.

However, this must be reinterpreted through mediCalm’s own identity.

### 6.1 The mediCalm Signature Layer
The app background should include a subtle visual field that evokes:

- topographic contour logic
- flowing water movement
- soft landscape memory
- micro-engraved precision
- organic but official design language

This layer should feel:
- faded but visible
- intricate but not distracting
- subconscious but not absent
- artistic but disciplined
- premium and engineered

### 6.2 Conceptual Inspiration
The background may abstractly echo:
- mountains
- rivers
- waterfalls
- shoreline calm
- horizon depth
- natural terrain

These references must never become literal illustrations.  
They must remain abstract, structural, and refined.

### 6.3 Psychological Intent
Nature is not decorative here.  
It is a subconscious reminder of:
- openness
- recovery
- regulation
- space
- healing environments
- safety beyond compression

---

## 7. Visual Depth Requirement

The page must include layered visual depth beyond standard flat UI.

It should carry a level of nuanced detail comparable in spirit to:
- microscale engraving logic
- subtle currency-style linework
- official security-pattern discipline
- high-end interface atmospherics
- intricate but restrained artistic layering

This does **not** mean ornamental clutter.  
It means **controlled, refined visual intelligence**.

The interface should reward attention at multiple scales:
- immediate readability from a distance
- soft emotional tone at medium view
- subtle detail and craftsmanship at close view

---

## 8. Background System Requirements

The homepage background should be treated as a designed system, not a generic fill.

### 8.1 Base Layer
A calm atmospheric gradient with restrained tonal variation.

Recommended mood:
- soft graphite
- desaturated teal
- muted slate
- soft mineral neutrals
- low-contrast cool-warm balance

Avoid:
- bright blue hospital tones
- bright green “nature app” tones
- purple meditation aesthetics
- high saturation of any kind

### 8.2 Mid Layer
Ultra-thin contour-inspired or terrain-inspired line structures.

Behavior:
- smooth
- sparse to moderate density
- gentle curvature
- non-chaotic
- no high-frequency repetition
- no aggressive intersections

### 8.3 Upper Layer
Very faint flow-field or waterline logic.

This may imply:
- river motion
- waterfall drift
- breath flow
- slow directional release

### 8.4 Opacity Discipline
All artistic background systems must remain subtle.

General guidance:
- texture and line systems should live in an extremely low-opacity range
- if the user consciously identifies the pattern immediately, it is probably too strong
- depth should be felt before it is analyzed

### 8.5 Motion
Motion must be minimal and calming:
- extremely slow drift
- soft pulsing
- no abrupt loops
- no energetic shimmer
- no visual noise

The background may gently synchronize with the app’s breathing logic, but this must remain understated.

## 9. Hero Section Requirements

The hero region must immediately anchor the experience.

### 9.1 Core Function
It must communicate:
- understanding
- authority
- calm
- intervention readiness

### 9.2 Tone
The tone must be:
- certain
- quiet
- professional
- compassionate without becoming sentimental
- grounded rather than dramatic

### 9.3 Suggested Headline Direction
The wording should center the idea that the user’s system is under tension or stuck in a protective state, and that mediCalm can guide it back toward baseline.

### 9.4 Supporting Copy
The supporting copy should mirror the user’s internal experience with medically realistic phrasing such as:
- rib restriction
- shallow breathing
- neck tension
- jaw tension
- protective overactivation
- current bodily intensity

The copy must avoid:
- sensationalism
- miracle language
- overclaiming
- spiritual vagueness

---

## 10. Input Layer Requirements

The homepage may collect a small amount of information before beginning the session, but this must remain friction-light.

### 10.1 Rule
The user must be able to begin relief quickly.

### 10.2 Current Required Direction
The interface should prioritize a minimal “current intensity” check-in.

This can be represented through a clean selection interface for present-state intensity.

### 10.3 Input Tone
Inputs should feel like calibration, not interrogation.

### 10.4 Long-Term Data Strategy Alignment
The current page should be designed with future engines in mind.

The homepage and immediate session-entry system should eventually help support:
- structured symptom stories
- better session tailoring
- history intelligence
- pattern recognition
- future adaptive engine development

But M3.4.1 should avoid overloading the user with too much input too early.

### 10.5 Design Principle
Collect only what is necessary for immediate usefulness.  
More advanced data collection can expand in later milestones.

- a personalized start

- calm

- trustworthy
- slightly luminous if appropriate

## 17. Implementation Guidance for Claude Code Sonnet 4.6


### 17.1 Mandatory Pre-Implementation Mockup Phase
Before writing implementation code for the final M3.4.1 homepage, Claude must first complete a structured concept exploration phase.

This phase is mandatory.

Claude must generate **4 distinct high-quality mockup directions** for the mediCalm front home page before implementation begins.

These 4 directions should all remain aligned with the same core product truth and identity pillars, but each should explore a slightly different emphasis in layout, visual atmosphere, hierarchy, and artistic background treatment.

Examples of differentiating axes may include:
- stronger official / precision-system emphasis
- stronger nature-coded atmospheric depth
- stronger medical-professional calibration
- stronger orb-centered regulation emphasis

These outputs must remain:
- modern
- medically grounded
- calming
- premium
- disciplined
- non-generic

They must not become random stylistic experiments disconnected from mediCalm’s actual psychological and product mission.

### 17.2 Mockup Evaluation Requirement
The 4 mockup outputs must be presented in a way that allows direct comparison before final implementation.

Each mockup should clearly communicate:
- visual direction
- structural hierarchy
- emotional tone
- background system behavior
- orb placement and role
- input treatment
- CTA character
- history section treatment

Claude should briefly identify the strengths of each direction and how each one interprets mediCalm’s identity.

### 17.3 No Final Implementation Before Selection
Claude must **not** jump straight into final implementation after the first concept.

The expected sequence is:

1. Produce 4 high-quality mockup directions
2. Compare them against the contract
3. Identify the strongest direction or best hybrid path
4. Only then proceed into implementation

### 17.4 Superpower Mockup Testing Requirement
If the development environment supports Superpower-style UI generation, advanced mockup assistance, or high-speed visual exploration tooling, that capability should be used in the mockup phase to maximize quality, variation, and speed.

The purpose of Superpower usage is not novelty.  
It is to accelerate top-tier concept testing before implementation lock-in.

### 17.5 Context7 Requirement
If Context7 is available in the development environment, it should be used to improve implementation grounding, design-system consistency, and technical confidence during both mockup generation and final build planning.

Context7 should help reinforce:
- implementation quality
- modern UI architecture awareness
- stronger consistency in component thinking
- higher confidence in design execution details

However, Context7 must not dilute the mediCalm-specific experience contract into generic health-app patterns.

### 17.6 Standard for Mockup Quality
The 4 pre-implementation outputs must be strong enough that any one of them could plausibly serve as the basis of a premium production direction.

These are not throwaway sketches.  
They are strategic visual candidates for a psychologically tuned homepage system.


Claude should interpret this as a build brief requiring:
- elite UX/UI craftsmanship
- production-capable structure
- disciplined aesthetic restraint
- subtle but meaningful artistic layering
- deep integration between visual design and user state

Claude should not default to generic health-app solutions.

Claude should design with:
- emotional precision
- modern product polish
- medically credible restraint
- subconscious trust signaling
- interface-level environmental intelligence

M3.4.1 succeeds if the homepage feels like:

> a modern, official, calming, medically grounded relief interface with intricate subconscious depth and a signature visual identity that invites immediate trust and engagement.

- a blank terminal
- a template dashboard
- a generic breathwork app
