---

You are implementing **M3.4.1** for **mediCalm**.

You must treat the loaded M3.4.1 homepage contract as the governing authority for design intent, UX psychology, visual discipline, tone, and implementation quality.

This is **not** a generic health-tech page, a meditation landing page, or a symptom tracker.

mediCalm’s front home page is the first nervous-system contact layer of the product.

Its job is to move the user through:

**Recognition -> Relief -> Action**

with a user who is likely arriving in active pain, protective tension, discomfort, uncertainty, or bracing.

---

## Primary Instruction

Do **not** immediately jump into final implementation.

Before implementation, you must complete a **mandatory mockup exploration phase**.

---

## Phase 1 — Pre-Implementation Mockup Testing

Produce **4 distinct homepage mockup directions** before final implementation.

These must all remain faithful to the M3.4.1 contract, while each exploring a different high-quality interpretation of the same system.

Each mockup must preserve:
- modern / ahead-of-its-time quality
- medically grounded professionalism
- calming psychological effect
- guided personalized relief framing
- subtle official-value trust signaling
- abstract nature-coded background depth
- premium interface polish

Each mockup should differ in a meaningful but disciplined way.

Good variation axes include:
- official precision-system emphasis
- contour / terrain background emphasis
- breath-orb-centered composition
- calmer medical interface emphasis
- stronger atmospheric depth and visual layering
- more structured history presentation
- different layout hierarchy while preserving the same core UX flow

These are not throwaway sketches.  
They must be legitimate premium candidate directions.

### For each of the 4 mockups, provide:
1. Mockup title
2. Core visual concept
3. Hero structure
4. Background system description
5. Orb behavior / placement
6. Input treatment
7. CTA treatment
8. Session history treatment
9. Why this direction works psychologically
10. Tradeoffs / risks

If possible in the environment, produce the mockups in a form that is highly legible to a developer and easy to compare side by side.

- premium and intentional

## Critical Visual Standard

The background should function like a mediCalm signature field:
- subtle
- layered
- abstract
- low-contrast
- nature-coded
- official-feeling
- intricate without distraction

It may draw structural inspiration from:
- topography
- river flow
- waterfall drift
- shoreline calm
- microscale engraved linework
- secure-document style pattern subtlety

But it must never become literal scenery or decorative illustration.

The effect should be subconsciously calming and trust-building.

---

## Critical UX Standard

The user should feel:
1. understood
2. safe
3. guided
4. confident that relief can begin now

The CTA must feel like entering a guided reset, not starting a task.

The orb must feel like a precision regulation instrument, not decoration.

Inputs must feel like calibration, not interrogation.

---

Your work should be top-tier, detailed, and implementation-ready.

You are expected to operate with elite UX/UI craftsmanship.

Every spacing decision, visual layer, wording choice, line weight, opacity decision, and motion behavior should contribute to psychological downshifting and trust.

Do not settle for generic.

Build for depth, precision, calm, and legitimacy.
