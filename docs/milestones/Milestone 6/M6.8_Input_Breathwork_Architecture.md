# M6.8 — Input → Breathwork Architecture (Claude Spec)

## Core Rule
NO direct input → ratio mapping.

Pipeline:
input → NeedProfile → FeasibilityProfile → BreathFamily → BreathPrescription → runtime

---

## NeedProfile

primaryNeed ∈ {
  downregulate,
  decompress,
  stabilize,
  activate
}

Rules:
- overwhelmed/anxious → downregulate
- pain/tight → decompress
- mixed/uncertain → stabilize
- sad/exhausted → activate OR stabilize (based on feasibility)

---

## FeasibilityProfile

Derived from:
- intensity
- sensitivity
- overload
- uncertainty
- lastOutcome

Rules:

if overload OR high sensitivity:
  - no counts
  - no rounds
  - no hold
  - short duration
  - low cognitive load

if uncertainty high:
  - use stabilize

if lastOutcome === worse:
  - reduce duration
  - simplify pattern

---

## BreathFamily

flare_safe_soft_exhale
decompression_expand
neutral_reset
calm_downregulate
gentle_activation

---

## Family Mapping

overwhelmed → flare_safe_soft_exhale
anxious → calm_downregulate
pain → decompression_expand
tight → decompression_expand
sad → gentle_activation OR neutral_reset
exhausted → gentle_activation OR neutral_reset
mixed → neutral_reset

---

## BreathPrescription

Fields:
- inhaleSeconds
- holdSeconds
- exhaleSeconds
- durationSeconds

Rules:

flare_safe_soft_exhale:
  3/5 or 4/6, no hold

decompression_expand:
  3/5 or 4/6 (never 2/4)

neutral_reset:
  3/5 or 4/6 or 3/3/6

calm_downregulate:
  4/6 or 4/7

gentle_activation:
  4/2 or 4/3 (short duration only)

---

## Global Constraints

- NO visible breath counts
- NO round counters
- progress = elapsedTime / durationSeconds
- single runtime clock only
- preserve orb UX
- no mechanical instructions

---

## UX Rules

- minimal language
- no performance framing
- no "complete X breaths"
- use soft phrasing

---

## HARI Context

Store:
- lastOutcome
- lastFamily
- lastDuration

Adjust:
- worse → simplify next session
- repeated failure → avoid same family

---

## Final Rule

All outputs must:
- respect feasibility
- match family intent
- reduce user effort
