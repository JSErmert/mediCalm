# M6.1.1 — SAD Safety Screen (v2.1)

We are updating M6.1.1 for mediCalm.

This update must remain fully consistent with M6.0, M6.1, and M6.2.

## Dependency

M6.2 must be implemented before this milestone to support pending state handling.

---

## Objective

Implement the SAD safety gate and escalation exit behavior triggered whenever the user selects **Sad**.

---

## Trigger

- Appears whenever **Sad** is included in `state_entry`
- Not intensity-gated
- Must occur before intake

---

## Safety Check Screen

### Title
Before we continue, a quick check

### Body
Are you feeling persistently low — in a way that goes beyond today?

### Actions
- **No, continue** (primary)
- **Yes** (secondary)

---

## Routing Logic

### If **No, continue**
- Proceed to intake
- Preserve all selected states

### If **Yes**
- Route to escalation exit
- Clear all pending session state

---

## Back / Dismiss Behavior

- Returns to `StateSelectionScreen`
- Preserves all selections including Sad

---

## Escalation Exit Screen

### Title
Pause here for a moment

### Body
This tool is designed for short-term regulation. What you're describing may benefit from more direct support.

If you can, consider reaching out to someone you trust or a professional resource.

### Actions
- Return Home
- View Support Resources

---

## Support Resources Screen

### Title
Support Resources

### Body
If you're in the U.S. and need immediate support, you can call or text **988** to reach the Suicide & Crisis Lifeline.

If you’re elsewhere, consider contacting a local crisis or mental health support service.

### Actions
- Return Home

---

## Support Resources Back Behavior

- Hardware/software back returns to the Escalation Exit Screen
- Does not return to intake or state selection

---

## State Handling

On **Yes**:
- `pendingStateEntry = null`
- No session continuation
- Clean return to home

---

## Tone Constraints

Must be:
- calm
- brief
- non-diagnostic
- non-judgmental

Must not:
- imply diagnosis
- pressure user
- drift into emotional coaching

---

## Identity Lock

mediCalm performs a boundary check, not a psychological evaluation.

---

## Status

M6.1.1 (v2.1) is implementation-ready.
