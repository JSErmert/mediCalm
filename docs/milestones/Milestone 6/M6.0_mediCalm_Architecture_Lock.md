# M6.0 — mediCalm Architecture Lock

## Definition

M6.0 defines mediCalm as a **state-aware physiological regulation system** where all user inputs are interpreted as expressions of autonomic dysregulation and routed through a single adaptive regulation engine.

---

## Core Principle (Non-Negotiable)

> One regulation engine. Multiple interpreted entry states.

* No separate systems per emotion
* No protocol fragmentation
* No emotional processing layer

All sessions resolve to:
→ reduce bracing
→ restore expansion
→ calm the system

---

## Identity Lock

mediCalm does NOT treat emotions.

mediCalm:
→ interprets user-selected states as physiological dysregulation
→ regulates the nervous system beneath the experience

Litmus test for all features:

> Does this interpret the state as autonomic dysregulation?

If not → reject

---

## State Entry Layer

User selects one or more:

* Pain
* Anxious
* Angry
* Sad
* Exhausted
* Tight
* Overwhelmed

Multi-select allowed.

---

## HARI Role (History-Adaptive Regulation Intelligence)

HARI interprets input as a **pattern**, not categories.

### Inputs

* Selected state(s)
* Intensity
* Sensitivity
* Body context (if applicable)
* Session history
* Known patterns

### Outputs

* Regulation bias
* Tone profile
* Breath modifier
* Effort level
* Session framing

---

## Multi-State Logic

> The user provides complexity. HARI removes it.

### Rules

* Never run multiple sessions
* Always collapse into one unified direction

### Classification

* 1–2 states → unified bias
* 3–4 states → prioritized classification
* 5+ states → Overload Protocol

---

## Conflict Resolution Hierarchy

Primary state defines constraints. Secondary states modify behavior.

Priority:

1. Overwhelmed → overrides complexity
2. Exhausted → overrides effort
3. Pain → overrides intensity
4. Anxious → controls pacing
5. Tight → modifies mechanics
6. Angry → modifies exhale force
7. Sad → modifies tone

## Safety Model

### Pain

Existing safety logic applies

### Sad (Critical)

Triggered on ANY selection

Pre-session screening:

> "Are you feeling persistently low or struggling in a way that feels hard to shift?"

If YES → escalate and exit
If NO → continue

Rule:

> mediCalm does not handle severe depressive states
