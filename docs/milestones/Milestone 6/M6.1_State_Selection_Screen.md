# M6.1 — State Selection Screen

## Purpose

Capture the user’s current internal state quickly and intuitively, without analysis, friction, or emotional framing drift.

This is the primary entry point into HARI’s state-aware regulation system.

---

## Core Principle

> The user selects how they feel.  
> HARI interprets it as physiological dysregulation.

---

## Screen Layout

### Header
Hi Josh. What are you feeling right now?

---

### Subtext
We’ll help your system settle.

---

## State Options

User can select one or more:

- Pain  
- Anxious  
- Angry  
- Sad  
- Exhausted  
- Tight  
- Overwhelmed  

---

## Interaction Model

### Multi-Select
- Enabled by default  
- No limit on selections  
- No prioritization required  

---

### Selection Behavior

**Unselected:**
- Neutral appearance  
- Low emphasis  

**Selected:**
- Soft highlight  
- No aggressive or alert colors  

---

### Continue Button

- Hidden until ≥1 selection  
- Appears immediately after selection  

Label:
Continue

---

## Routing Logic

On Continue:


state_entry = [selected states]
selection_count = n


### Conditional Flow

- If `Sad` is selected → route to **M6.1.1 SAD Safety Screen**
- Else → proceed to **Intake Screen**

---

## Multi-State Handling (HARI)

Handled internally. Not shown to user.

- 1–2 states → unified interpretation  
- 3–4 states → prioritized classification  
- 5+ states → Overload Protocol  

---

## UX Constraints

### Do NOT:
- Explain categories  
- Define emotional states  
- Ask follow-up questions  
- Force prioritization  
- Show system logic  

---

### MUST:
- Enable instant recognition  
- Allow fast selection  
- Maintain simplicity  
- Preserve physiological framing  

---

## Optional Micro-Enhancement

Secondary descriptors (only if needed for clarity):

- Anxious *(on edge / uneasy)*  
- Tight *(body feels tense)*  
- Overwhelmed *(too much at once)*  

Use only if it does not clutter the interface.

---

## Edge Cases

### No Selection
- Continue button remains hidden  

---

### All States Selected
- No restriction  
- No warning  
- Handled later via Overload Protocol  

---

## Framing Integrity

This screen must not:
- Diagnose  
- Interpret  
- Analyze  

It only captures user input.

---

## Definition

> M6.1 State Selection is a neutral, fast-entry interface that allows users to express their current state while preserving mediCalm’s identity as a physiological regulation system.
