# M4.7.1 — Body Context UI System Specification (Final)

## Status
Production-grade UI + interaction specification.

This is the **single source of truth** for BodyContextScreen.

This spec must be followed exactly for implementation.

---

## 1. Purpose

BodyContextScreen allows users to:
- view
- add
- edit
- remove

persistent context in a way that is:
- calm
- lightweight
- non-clinical
- scalable over time

---

## 2. Core UX Principle

> This is a personal context layer, not a medical form.

Must feel like:
- notes
- memory
- pattern awareness

Must NOT feel like:
- intake form
- questionnaire
- diagnosis tool

---

## 3. Screen Layout

### Top → Bottom
