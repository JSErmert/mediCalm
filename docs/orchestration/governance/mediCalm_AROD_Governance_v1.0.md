# mediCalm AROD Governance v1.0

## Document Type
Internal realism, safety, and truth-discipline governance

## Project
mediCalm

## System
AROD (adapted)

## Purpose
This document governs truth-status discipline, realism pressure, contradiction detection, and claim containment in mediCalm.

It exists to prevent mediCalm from becoming:
- elegant but overconfident
- adaptive but unsafe
- calming in tone while structurally misleading
- persuasive beyond what the system actually knows

## Core Principle

> A calm-feeling system is not a trustworthy system unless its limits are explicit.

---

## 1. What AROD Governs in mediCalm

AROD governs:

- truth-status discipline
- claim containment
- contradiction detection
- realism pressure on recommendations
- safety language restraint
- adaptation honesty
- interpretation integrity

It does not govern:
- milestone sequencing
- cue timing
- visual layout
- branch orchestration

---

## 2. Truth-Status Model

mediCalm must distinguish clearly between:

- confirmed
- inferred
- suggested
- weak
- unresolved
- blocked by safety
- execution-ready

The system must never blur:
- user-stated state
- system interpretation
- memory-informed pattern
- proven helpfulness
- temporary preference

Preference is not proof.
Pattern is not certainty.
Adaptation is not validation.

---

## 3. Claim Containment Rules

mediCalm must not imply stronger certainty than it has.

Do not overclaim:
- what a breathing pattern will do
- what a body state “means”
- what a learned pattern proves
- that a session is medically corrective
- that a user preference equals a validated intervention

Use restrained framing such as:
- may help
- tends to
- suggests
- keep gentle for now
- based on current inputs

Avoid false confidence.

---

## 4. Contradiction Detection

AROD review must catch contradictions such as:

- activation bias during overload or sensitivity
- calming language paired with effort-heavy delivery
- fast-entry logic that bypasses feasibility
- refinement choices that conflict with safety gates
- learning logic that treats one-off preference as truth
- copy that implies treatment certainty
- guidance that sounds soothing but increases user pressure

Contradictions must be surfaced early.

---

## 5. Safety Realism

If the system faces a tension between:
- user preference
- adaptive convenience
- calm UX
- safety realism

safety realism wins.

Examples:
- do not expose options that are not actually allowed
- do not let refinement bypass feasibility
- do not let “continue what helped” drift too far from validated base logic
- do not use language that implies clinical certainty beyond the system’s scope

---

## 6. Interpretation Integrity

mediCalm may interpret, but must not hallucinate confidence.

Interpretation must remain bounded:
- state selection informs need
- intake informs feasibility
- patterns may influence delivery
- memory may bias future defaults
- none of this becomes absolute truth without repeated evidence

The system should be useful without pretending omniscience.

---

## 7. Learning Discipline

M8 and future adaptive layers must obey these rules:

- repeated signal > one-off signal
- consistent context > isolated outcome
- bounded adjustment > sweeping change
- user preference != measured effectiveness
- silence about uncertainty is not allowed in architecture decisions

Learning must remain slower than temptation.

---

## 8. Copy Discipline

Language in contracts, prompts, and product copy should remain:

- careful
- non-overclaiming
- low-drama
- grounded
- specific when possible
- modest when certainty is limited

Do not write copy that flatters the system into sounding more validated than it is.

---

## 9. AROD Review Outputs

AROD review may return:

- safe and grounded
- overclaimed
- weakly supported
- contradictory
- needs tighter truth-status language
- unrealistic adaptation proposal
- acceptable only with clamping / gating
- defer until evidence or structural support improves

---

## 10. Failure Conditions

AROD governance has failed if mediCalm begins to:

- overstate effectiveness
- imply clinical certainty it does not possess
- confuse “helped once” with validated intervention
- hide contradictions for simplicity
- adapt too strongly from weak evidence
- let refinement or convenience outrun safety truth

---

## 11. Final Rule

If a stronger-sounding claim improves polish but weakens honesty, honesty wins.

mediCalm should feel:
- calm
- serious
- bounded
- trustworthy
- explicit about uncertainty where needed
