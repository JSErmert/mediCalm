protocol is directly proven by a single publication. Instead, it does

MediCalm, 2. records validated bibliographic/reference details that can

distinguishing:  - \`source_grounded\`  - \`product_inference\`  -
\`validation_needed\`

\## 6. What Is Not Yet Safe to Claim as Publication-Grounded

The following should \*\*not\*\* be labeled as directly source-grounded
unless you later validate exact page-level or publication-level support:

\- "Posterior rib expansion should always come before cervical
reconnection." - "A 4-in / 7-out breath ratio is best medical practice
for this symptom cluster." - "MediCalm's sequence engine reflects
established medical protocol ordering." - "The app's reinforcement
learning strategy is medically validated." - "The exact UI reassurance
wording is based on medical publications."

These may still be good product ideas, but right now they should remain
labeled as either:

\- \`product_inference\` - \`design_decision\` - \`validation_needed\`
