# MediCalm Visual Design Doctrine
Version: v1
Status: Draft
Owner: Josh
Depends on: Product Vision, Guided Session UI Spec, Safety + Reassurance Spec

## Purpose

This document defines how MediCalm should look and feel visually so that users experience it as calm, professional, clean, and medically reassuring.

## Visual Goal

The interface should feel:
- clinically calm
- modern and minimal
- trustworthy
- spacious
- steady
- soft but professional

## Visual Anti-Goals

The interface should not feel:
- flashy
- toy-like
- hyper-wellness branded
- high-energy
- coldly sterile
- cluttered
- gamified during pain

## Design Principles

1. Calm over stimulation
2. Clarity over decoration
3. Reassurance over novelty
4. Spaciousness over density
5. Smoothness over theatrical motion

## Color Logic

Recommended palette direction:
- soft neutrals
- muted blues
- gentle greens
- warm off-whites

Avoid:
- neon accents
- harsh red dominance
- over-saturated gradients

## Typography Standard

Typography should be:
- readable
- stable
- clean
- non-playful
- comfortable on phone and desktop

## Motion Doctrine

Animations should be:
- smooth
- slow enough to regulate
- performance-conscious
- non-jittery
- emotionally quiet

## Layout Doctrine

The UI should use:
- generous whitespace
- simple hierarchy
- large touch targets
- centered breathing visuals during sessions
- low text density during active guidance

## Cross-Platform Standard

The system should remain visually coherent across:
- desktop browsers
- iPhone/mobile layouts

Responsive rules should prioritize:
- readable timers
- large feedback buttons
- centered visual pacing
- one-handed usability where possible
