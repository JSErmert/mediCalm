# MediCalm Glossary
Version: v1
Status: Draft
Owner: Josh
Depends on: All core docs

## Purpose

This document locks canonical meanings for MediCalm terms to prevent implementation drift.

## Canonical Terms

### Source Truth
A distilled, structured statement derived from trusted source domains.

### Mechanism
A functional explanation the system uses to interpret what may be happening in the body.

### Symptom Pattern
A structured pattern of user-reported experience mapped to candidate mechanisms.

### Protocol Component
The smallest reusable intervention unit, such as a breath cue, posture cue, or micro-movement.

### Protocol Definition
An ordered sequence of protocol components with timing, purpose, and boundaries.

### State Intake
The process of capturing the user’s current symptom state.

### State Signature
A normalized representation of a repeated state pattern used for learning.

### Follow-Up Protocol
A second protocol offered only when conditions indicate it is safe and useful.

### Safety Override
A runtime rule that blocks, stops, or redirects a protocol flow for safety reasons.

### Flare
A state of symptom amplification, increased guarding, overload, or nervous-system tightening.

### Source-Grounded
A claim or logic element directly supported by trusted source domains.

### Product Inference
A design-level interpretation built on source-grounded material but not directly quoted from it.

### Design Decision
A product or interface choice made for usability, clarity, or implementation quality.

### Validation Needed
A statement or idea that may be promising but still requires further evidence, testing, or review.
