## 3. Behaviors Implemented

### M5.1 — Pattern Detection

- 7 dimensions: symptom_focus, flare_sensitivity, session_intent, session_length, protocol_benefit, pacing, outcome_trajectory
- State machine: `candidate` (≥2) → `active` (≥3 + weighted score) → `high_confidence` (≥5 + clean window) / `suspended`
- Recency weighting: positional decay 0.80^n, 90-day age cap at 0.25, staleness floor 0.15
- Contradiction handling: standard ×2 penalty, safety ×3 penalty; position 0/1 = recent_contradiction flag → state capped
- Flare safety exception: `high` observations always weighted ≥ 0.50; toward-higher = safety-level event
- D1 mitigation: `pacing_tendency` capped at `candidate` (never eligible for M5.2/M5.3) until round_plan migrated
- Cache: localStorage key `hari_pattern_summary`, invalidated on save/delete/validate via existing M4.7 hooks
- Graceful degradation: any error → all-null patterns, schema_version mismatch → recompute

### M5.2 — Adaptive Intake Defaults

- Pre-populates `session_intent`, `symptom_focus`, `flare_sensitivity`, `session_length_preference`
- Eligibility: `active` or `high_confidence` state only; `candidate` and `suspended` excluded
- Per-field `{ value, pattern_state }` output — each field independent; absent = no suggestion
- Flare safety floor: `flare_sensitivity` suggestion never lower than most recent validated observation; `not_sure` excluded from ordinal comparison
- Suggestion indicator: `.chipSuggested` (dashed border) + `.suggestionLabel` per field
- User override: `clearSuggestion(field)` removes indicator on first interaction; no lock-in
- Graceful degradation: empty `{}` on any failure → standard intake behavior unchanged

### M5.3 — Protocol Hint Reinforcement

- Reads `protocol_benefit_tendency` → `active` produces `soft` hint, `high_confidence` produces `moderate`
- Safety suppression (any one condition → `suppressed_by_safety: true`, no hint applied):
  - Protocol pattern `suspended`
  - `outcome_trajectory.dominant_value === 'adverse'`
  - `flare_sensitivity_tendency.has_recent_contradiction` or `suspended`
- Flare constraint: eligible `high` flare tendency caps hint at `soft` (never increases aggressiveness)
- Integration: hint passed as optional param to `selectIntervention()`; applied in `resolveHintedProtocol()` only
- Conservativeness rank gate: hint rejected if it would make the session less conservative than M4.5 determined
  - PROTO_REDUCED_EFFORT (0) ≤ PROTO_CALM_DOWNREGULATE (1) ≤ PROTO_STABILIZE_BALANCE (2)
  - Hint only accepted if `rank(hinted) ≤ rank(m45Selected)`
- Pacing advisory: applied after all M4.6 safety checks; never `extended`; D1 means always absent in practice
- Graceful degradation: any failure → `{ hint_strength: 'none', suppressed_by_safety: false }`

### M5.4 — Session Insights Surface

- Surfaces up to 3 non-diagnostic insights on `SessionSetupScreen` below session info
- Dimension order: flare_sensitivity → symptom_focus → session_length → session_intent → outcome_trajectory
- Excluded: `protocol_benefit_tendency` (exposes internal IDs), `pacing_tendency` (D1 debt, technical values)
- `outcome_trajectory` filtered to `adverse`/`improving` only; `stable`/`variable` omitted
- Sort: `suspended` first (safety-relevant), then `high_confidence` → `active` → `candidate`
- State display: `candidate` → "(early signal)" label; `suspended` → "(paused)" label (M5.4 §4)
- Renders nothing when `insights.length === 0` — zero layout impact
- Graceful degradation: any failure → `[]` → panel hidden
