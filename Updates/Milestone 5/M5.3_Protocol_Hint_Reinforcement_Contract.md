# M5.3 — Protocol Hint Reinforcement Contract

## Status
Definition layer for bounded protocol hint adaptation.

This contract defines how HARI may use validated pattern signals from M5.1 to softly reinforce protocol preference inside the existing M4/M5 system.

M5.3 is a recommendation-weighting layer only.

It does NOT:
- create new protocol classes
- replace intervention selection
- bypass safety logic
- force protocol choice
- introduce hidden adaptation

---

## 1. Purpose

M5.3 exists to:

- reinforce protocol hints when validated evidence supports them
- improve initial fit of protocol selection
- preserve existing intervention architecture while allowing bounded learning

This layer should make HARI more useful over time without making it opaque or overconfident.

---

## 2. Core Principle

> M5.3 may reinforce, but never dictate.

Protocol hints must remain:
- advisory
- bounded
- explainable
- reversible
- safety-constrained

The existing M4.5 intervention selector still owns the final recommendation logic.

---

## 3. Input Source

M5.3 reads only from:

```ts id="p0c7w4"
getOrComputePatternSummary()

and only through approved dimensions from M5.1.

No raw history access is permitted.

4. Eligible Pattern Dimensions

M5.3 may use only:

4.1 protocol_benefit_tendency

Primary allowed dimension.

Used to softly reinforce a previously beneficial mapped protocol when evidence is strong enough.

4.2 pacing_tendency

Secondary allowed dimension.

Used only to advise round/pacing preference within already approved M4.6 bounds.

4.3 flare_sensitivity_tendency

Constraint-only dimension.

May reduce reinforcement confidence or suppress reinforcement when recent validated evidence suggests higher sensitivity.

This dimension must never be used to make protocol hints more aggressive.

5. Pattern Eligibility Rule

A pattern may influence protocol hints ONLY if:

state ∈ { 'active', 'high_confidence' }
state ≠ 'suspended'
no recent safety contradiction is active

Otherwise:

ignore the pattern completely

Candidate patterns may not influence protocol hints.

6. Protocol Hint Scope

M5.3 may only influence:

the preferred ordering among already approved mapped protocols
a soft hint into existing hariHintProtocolId
pacing advisory within existing round-size limits

M5.3 must NOT:

create a new protocol
create a new intervention class
bypass M4.5 selector constraints
override current-session-first logic
suppress safety downshifting
persist hidden protocol preferences outside approved storage
7. Reinforcement Behavior
7.1 Soft Reinforcement Only

If protocol_benefit_tendency is eligible, M5.3 may produce:

ProtocolHintReinforcement {
  hinted_protocol_id?: string
  hint_strength: 'none' | 'soft' | 'moderate'
  pacing_advisory?: RoundCountProfile
  source_pattern_state: PatternState
}

Hint strength must remain bounded.

Recommended rule:

active → soft
high_confidence → moderate

No stronger hint level is allowed in M5.3.

7.2 No Forced Selection

Even if a hint exists:

M4.5 must still evaluate current session state
safety constraints must still dominate
the hint may be ignored if current state conflicts with it

A historical beneficial protocol must never override present-session worsening, fragility, or uncertainty.

7.3 Safety Suppression Rule

If recent evidence includes:

adverse protocol outcome
worsening trend
suspended protocol pattern
elevated flare contradiction

Then:

protocol hint reinforcement must be suppressed entirely

Safety contradictions cancel reinforcement.

8. Pacing Advisory Rule

If pacing_tendency is eligible:

it may advise minimal, short, or standard
it may NOT expand beyond existing M4.6 safe round limits
it may NOT force longer rounds under fragile current-state conditions

Pacing advisory is subordinate to:

current flare sensitivity
current session tolerance
reassessment urgency
current-session safety logic
9. Explainability Requirement

M5.3 must support explanation such as:

"A protocol similar to this has helped in several recent validated sessions, so it is being suggested again."

and, when suppressed:

"A past preference exists, but current session safety takes priority."

No hint may be applied if it cannot be explained in plain language.

10. Failure Modes to Prevent
A — Historical override

Past helpful protocol overrides current unsafe conditions

B — Hidden reinforcement

Protocol is biased without visible or explainable basis

C — Aggressive escalation

High-confidence history increases intensity beyond safe bounds

D — Pattern drift

Weak or contradictory protocol evidence still influences hints

E — Cross-layer leakage

M5.3 modifies intervention selection logic outside allowed hint fields

11. Graceful Degradation

If:

PatternSummary is unavailable
no eligible protocol pattern exists
pacing pattern is candidate / insufficient / suspended
contradiction is active

Then:

return hint_strength: 'none'
apply no pacing advisory
fall back entirely to M4.5 + M4.6 baseline behavior
12. Integration Rules

M5.3 must integrate only through approved hint channels such as:

hariHintProtocolId

and equivalent bounded advisory inputs.

It must NOT:

rewrite intervention selection logic
change protocol definitions
alter persistence schema
modify validation behavior
13. Output Contract

M5.3 must produce a structured output:

interface ProtocolHintReinforcement {
  hinted_protocol_id?: string
  hint_strength: 'none' | 'soft' | 'moderate'
  pacing_advisory?: RoundCountProfile
  source_pattern_dimension?: 'protocol_benefit_tendency' | 'pacing_tendency'
  source_pattern_state?: PatternState
  suppressed_by_safety: boolean
  plain_summary?: string
}

This output is advisory only.

14. User Control Requirement

M5.3 must not reduce user control.

If future surfaces expose protocol hints:

users must still be able to proceed normally
users must not be locked into hinted pathways
hints must remain suggestions, not instructions

Even when not surfaced directly, the system must remain structurally reversible.

15. Final Instruction to Claude Code

Implement M5.3 as a bounded protocol hint reinforcement layer.

Do NOT:

create new interventions
bypass current-session-first logic
override safety constraints
force protocol selection
access raw history

Ensure:

reinforcement is soft and reversible
current safety always wins
pacing advisory stays within existing M4.6 limits
all effects remain explainable
16. System Boundary Reminder

M5.3:

reads eligible pattern summaries
emits bounded protocol and pacing hints

M5.3 does NOT:

make final session decisions
modify M4 safety behavior
create new treatment logic
infer diagnosis
End of Contract
