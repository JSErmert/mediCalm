# M4.7.2 — Session Completion UX & Flow Integrity Contract

## Status
Implementation contract for M4.7.2.

This contract corrects the session completion flow and removes redundant or conflicting decision layers.

This ensures:
- a clear user mental model
- a single persistence point
- consistent session lifecycle behavior

---

## 1. Purpose

M4.7.2 exists to ensure:

> session completion and session saving are separate, explicit steps

This prevents:
- duplicate decision layers
- accidental persistence
- user confusion

---

## 2. Core Principle

> Ending a session is NOT the same as saving a session

The system must:
- allow the user to stop a session
- then decide whether to save or discard
